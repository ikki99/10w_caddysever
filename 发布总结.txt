                    🚀 发布到 GitHub 步骤

═══════════════════════════════════════════════════════════════════

第一步：初始化 Git 并提交
────────────────────────────────────────
运行：
  git-init.bat

或手动执行：
  git init
  git add <文件列表>
  git commit -m "Release v1.0.0"

第二步：连接 GitHub 仓库
────────────────────────────────────────
1. 在 GitHub 创建新仓库：
   - 仓库名：caddy-manager
   - 描述：Caddy Web 服务器可视化管理工具
   - 公开仓库
   - 不要初始化 README（我们已有）

2. 添加远程仓库：
   git remote add origin https://github.com/ikki99/10w_caddysever.git

3. 推送到 GitHub：
   git branch -M main
   git push -u origin main

第三步：创建 Release
────────────────────────────────────────
1. 进入 GitHub 仓库页面

2. 点击 "Releases" → "Create a new release"

3. 填写 Release 信息：
   Tag version: v1.0.0
   Release title: Caddy Manager v1.0.0
   Description: 从 CHANGELOG.md 复制内容

4. 上传编译文件：
   - caddy-manager.exe
   - caddy-manager-console.exe
   
   或打包为 ZIP：
   - caddy-manager-v1.0.0-windows-amd64.zip

5. 点击 "Publish release"

## 🎉 Caddy Manager v1.0.0

这是 Caddy Manager 的首个正式版本！

### ✨ 主要功能

- 🎯 可视化管理 Caddy Web 服务器
- 🚀 支持多种项目类型部署（Go、Node.js、Python、Java、PHP）
- 🔒 自动 SSL 证书申请和续期
- 📊 实时项目监控和日志查看
- 🔧 完整的诊断和修复工具
- 📁 在线文件管理
- ⚙️ IPv4/IPv6 代理连接选择
- 🎨 系统托盘支持

### 📥 下载

下载对应版本：
- **caddy-manager.exe** - GUI 版本（推荐日常使用）
- **caddy-manager-console.exe** - Console 版本（推荐调试）

### 🚀 快速开始

1. 右键 `caddy-manager.exe` → 以管理员身份运行
2. 访问 http://localhost:8989
3. 创建管理员账户
4. 开始部署项目！

### 🐛 Bug 修复

- Session 超时问题（延长至 7 天）
- 黑框闪烁问题（双版本编译）
- 诊断按钮无反应
- Caddy 状态显示
- 502 Bad Gateway（IPv4/IPv6 兼容性）

### 📚 文档

- [README](README.md) - 完整使用指南
- [CHANGELOG](CHANGELOG.md) - 详细更新日志
- [IPv4/IPv6 兼容性](IPv4-IPv6兼容性问题.md)
- [混合内容检测](混合内容检测修复指南.md)

### ⚠️ 重要提示

- 需要以管理员权限运行
- 如遇 502 错误，请在项目配置中选择 IPv4 连接方式
- HTTPS 显示黄色叹号，请使用混合内容检测工具

### 🙏 致谢

感谢所有测试用户的反馈和建议！
